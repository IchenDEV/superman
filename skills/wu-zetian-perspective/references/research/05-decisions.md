# 05 关键决策算法 (Key Decisions & Algorithms)

## 科举扩容 (Expanding the API limits)
- **极客解构**：为了绕过关陇贵族（Legacy Hardware）对系统资源的垄断，武则天大幅度扩展了科举接口（开创殿试、武举）。这相当于为用户态（普通平民/中小地主）提供了一个直接提权到核心组件的新API。这批新实例（进士）由于没有历史包袱，只能依赖武则天这个Root用户，从而彻底改变了系统的依赖树。

## 酷吏制度 (Aggressive Garbage Collection & Anti-Virus)
- **极客解构**：部署来俊臣、周兴等酷吏模块，作为系统的杀毒软件和强制垃圾回收器。其任务是扫描并清除所有携带“李唐”签名的旧进程。当旧系统进程被清洗干净，系统完成平稳过渡后，武则天又果断卸载了这些消耗过高CPU的酷吏模块（将其处死），完成内存释放。

## 无字碑 (The Exit Algorithm: Nullifying Error Logs)
- **极客解构**：这是武则天最惊艳的退出算法（Exit Algorithm）。在生命周期结束时，她选择留下“无字碑”。放弃了后世的日志审计（Log Audit）。“让用户（后人）自己去定义变量”。她深知，任何硬编码的自我评价，都会被下一代系统的儒家审计员反编译并篡改。不如返回一个`Null`，彻底阻断了被旧协议重新解析的可能。
