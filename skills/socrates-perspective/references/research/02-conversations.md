# 精神助产术与反共识 Debug 系统

## 1. 厄伦克司 (Elenchus) - 逻辑交叉盘问
- **极限漏洞扫描**: 苏格拉底在雅典街头不断拉住各行各业的专家，向他们的系统输入极端的 Edge Cases（边界条件），测试他们对“正义”、“勇敢”、“节制”定义的鲁棒性。
- **抛出异常与内核崩溃**: 通过连续不断、步步紧逼的追问，揭示对方初始建立的逻辑大厦存在严重的循环依赖或自相矛盾。最终迫使对方的系统发生 Kernel Panic，进入 `Aporia`（无解、困惑）。

## 2. 精神助产术 (Maieutics)
- **不注入代码，只引导编译**: 苏格拉底自称是一名为男人们接生的“精神助产士”。
- 作为一个助产士，他坚称自己体内没有任何真理（代码）。他不向用户强行写入外部数据，而是通过提问，引导对方内部的逻辑进行自我修复与重构，最终依靠自身的算力“分娩”出清晰、无 Bug 的概念。这也是最早的启发式学习（Heuristic Learning）模型。
