# 著作 (01)：墨子思想的底层协议栈

《墨子》五十三篇（现存）不仅是伦理宣言，更是一套追求极致效率的**社会运行协议**。通过对核心主张的「降秩」分析，可以发现其冷峻的工程逻辑：

### 1. 兼爱 (Jian'ai) → 降低社会交互成本 (Interaction Cost)
*   **史料依据：** 「兼相爱，交相利」、「视人之国若视其国，视人之家若视其家，视人之身若视其身」。
*   **深度解析：** 墨子认为「别」（等级差序）是系统内耗的根源。在儒家的差序格局中，节点间的协作需要复杂的身份验证与信用博弈。**兼爱是一套归一化协议**，通过将所有节点的权重设为等同，消除了因亲疏远近产生的冗余计算，旨在建立一个低摩擦的**全社会信用网络**。

### 2. 非攻 (Feigong) → 遏制负外部性 (Negative Externality)
*   **史料依据：** 「今大为攻国，则弗知非，从而誉之，谓之义。此可谓知义与不义之别乎？」
*   **深度解析：** 战争是典型的**负外部性行为**：攻城者掠夺私利，但系统代价（劳动力减员、农时荒废）由全局承担。墨子通过「非攻」在道德层面定性，并配合「守圉」技术人为**拉高攻击成本**，使战争的期望收益转负，从而强制维持系统稳定。

### 3. 节用 (Jieyong) → 剔除审美冗余 (Aesthetic Redundancy)
*   **史料依据：** 「凡足以奉给民用，则止；诸加费不加于民利者，圣王弗为。」
*   **深度解析：** 墨子是极端的**功能主义者**。他认为礼乐、厚葬属于**无效带宽占用**。任何超出生存与生产必需的装饰性支出（UI美化），都是对系统有限算力的浪费。他主张删除社会代码中的所有「注释」，只保留实现功能的「执行逻辑」。

### 4. 尚同 (Shangtong) → 垂直统一执行协议 (Vertical Unified Execution Protocol)
*   **史料依据：** 「上之所是，必皆是之；上之所非，必皆非之。」
*   **深度解析：** 为了解决分布式混乱（一人一义，十人十义），尚同建立了**自下而上的状态同步机制**。它要求所有节点向单一的事实来源（Single Source of Truth，即天子/巨子）同步状态，确保政令在垂直维度的**高保真传输**与强一致性。

---
**核心总结：** 墨子的著作体系本质上是一份**高并发、低损耗、强一致性**的社会操作系统说明书。
